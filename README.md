### 2. Lab 2 — POT ADC Read
การทดลองอ่านสัญญาณอนาล็อก (Analog-to-Digital Converter) จาก **Potentiometer (ตัวต้านทานปรับค่าได้)** โดยต่อขากลางเข้ากับ GPIO 34 (ADC1 Channel 6) เพื่อตรวจจับค่าความต่างศักย์ไฟฟ้าที่แปรผัน และส่งผลลัพธ์ผ่านโปรโตคอล Serial ออกมาแสดงผลบน Serial Monitor 

* **เทคโนโลยีและคุณสมบัติของระบบ:**
  * **API ที่ใช้ควบคุม:** `esp_adc/adc_oneshot.h` (รองรับมาตรฐานใหม่บน ESP-IDF)
  * **ความละเอียดของสัญญาณ:** 12-bit (ให้ค่าสัมบูรณ์ในช่วง 0 – 4095)
  * **ค่า Attenuation:** `ADC_ATTEN_DB_12` (รองรับช่วงแรงดันสูงสุดที่ 0V – 3.3V)
* **ซอร์สโค้ด:** [main.c](file:///Users/phuwish/IOT%20Subject/Week-02-Sensors-and-calibration/ESP32_Project/POT_ADC_Read/main/main.c)
* **วิดีโอบันทึกผลการทดลอง:** [รับชมวิดีโอ](https://youtube.com/shorts/vc_8T5pwhvw)

### 3. Lab 3 — POT Angle Measure (พร้อมระบบคาลิเบรต)
การนำค่า ADC ที่อ่านได้มาทำการประมวลผลต่อเพื่อวัดมุมองศาการหมุนจริงของ Potentiometer (0° – 180°) โดยการใช้เทคนิคทางซอฟต์แวร์เข้ามาปรับจูนค่าสัญญาณเพื่อแก้ปัญหา **Non-linearity** (ความไม่เป็นเส้นตรงทางกายภาพของตัวอุปกรณ์) 

* **ซอร์สโค้ด:** [main.c](file:///Users/phuwish/IOT%20Subject/Week-02-Sensors-and-calibration/ESP32_Project/POT_Angle_Measure/main/main.c)
* **วิดีโอบันทึกผลการทดลอง:** [รับชมวิดีโอ](https://youtube.com/shorts/r8VO6wkyAUE)

#### ⚙️ เทคนิคหลักที่เลือกใช้ในการแก้ปัญหา
| เทคนิคที่ใช้ | วัตถุประสงค์และกลไกการทำงาน |
| :--- | :--- |
| **Piecewise Linear Interpolation** | แบ่งช่วงมุมวัดออกเป็น 4 ช่วงย่อย ($0 \rightarrow 45 \rightarrow 90 \rightarrow 135 \rightarrow 180^{\circ}$) แล้วใช้สูตรคณิตศาสตร์เชิงเส้นคำนวณแยกส่วนกันในแต่ละช่วงเพื่อชดเชยค่าเบี่ยงเบนของอุปกรณ์ |
| **Moving Average Filter** | ประยุกต์ใช้การอ่านค่าเฉลี่ยสะสม 32 ครั้งก่อนนำมาคำนวณ เพื่อทำหน้าที่ลดสัญญาณรบกวน (Noise Filter) ทำให้ค่ามุมที่อ่านได้มีความนิ่งและเสถียร ไม่เกิดอาการแกว่งไปมา |
| **Interactive Self-Calibration** | คาลิเบรตสดผ่าน Serial Interface ทุกครั้งหลังระบบบูต โดยผู้ใช้งานหมุนแกน POT ไปยังมุมอ้างอิงบนกระดาษ แล้วกด Enter เพื่อให้ระบบเก็บแรงดันจริง ณ ขณะนั้น แก้ปัญหาความคลาดเคลื่อนจากการยึดอุปกรณ์กายภาพ |

### 📊 ตารางบันทึกผลการคาลิเบรต (ข้อมูลจริงจากการทดลอง)
เมื่อทดสอบการวัดค่าเปรียบเทียบในระบบ **Interactive Mode** ได้ผลลัพธ์ข้อมูลการจับคู่ค่ามุมกายภาพกับค่าสัญญาณดิจิทัล ดังนี้:

| มุมกายภาพ (องศา) | ค่า ADC เฉลี่ย (64 ตัวอย่าง) |
| :---: | :---: |
| $0^{\circ}$ | 1194 |
| $45^{\circ}$ | 198 |
| $90^{\circ}$ | 144 |
| $135^{\circ}$ | 14 |
| $180^{\circ}$ | 3 |

💡 **ข้อสังเกตและวิเคราะห์ผล:** 
จากการทดลองพบว่า ค่า ADC ที่บันทึกได้จริงจะมีทิศทางตรงกันข้าม (กลับทิศ) กับสมมติฐานทางทฤษฎีในใบงาน (กล่าวคือ ค่า ADC จะมีค่าลดลงเมื่อองศาเพิ่มขึ้น) ซึ่งเป็นผลมาจากการเลือกขั้วสายไฟฝั่งบวกและฝั่งกราวด์ของตัวต้านทานปรับค่าได้สลับด้านกัน อย่างไรก็ตาม **ระบบที่เขียนไว้ไม่เกิดข้อผิดพลาด** เนื่องจากโค้ดอัลกอริทึม Piecewise Interpolation ที่ออกแบบขึ้น สามารถรองรับและคำนวณทิศทางของค่าความชัน (Slope) ได้โดยอัตโนมัติทั้งสองทิศทาง
